Core Capabilities

Built for Absolute Autonomy

OpenDroid bypasses simple prompt-response chatbot models to deliver a real agentic automation system that acts on your behalf.

Autonomous Planning

Decomposes complex requests into multiple distinct execution actions. Validates progress and adapts planning steps dynamically when system states change.

Vision & Screenshot OCR

Captures screen buffers via accessibility layers and leverages multimodal analysis. Automatically falls back to layout tree scraping when screen capture is restricted.

Multi-Tier Memory

Maintains Working, Episodic, Semantic, and Procedural layers. Structured personal facts are extracted via an LLM fact-mining parser to keep queries grounded.

Offline Voice Processing

Uses local wake-word detection and local speech recognition engines. Translates voice commands quickly, falling back to ElevenLabs when cloud services are desired.

System Intent Safeguard

Ensures zero-refusal messaging and calling by defaulting to background services. Seamlessly launches system composer fallbacks if standard permissions are missing.

Procedural Macro Engine

Records sequential device operations and registers custom workflows. Macros can be scheduled using standard cron expressions to run tasks automatically.

Inside OpenDroid

System Architecture Flow

Designed with clean architecture principles, utilizing Dagger-Hilt for system dependency orchestration, local database caching, and accessibility automation loops.

INPUT

Voice / Chat Interface

Ingests local sound waves, wakes the agent offline, or processes text strings.

PLANNER

LLM Planner & Re-Evaluator

Evaluates context intents, compiles execution tasks, and validates results post-action.

MEMORY

Multi-Tier Database Repository

Resolves preferences via Room DB, updates history indices, and fetches macros.

DISPATCHER

Action Dispatcher

Instructs system controls and schedules tasks via fallback-safe system triggers.

AUTOMATION

Accessibility & Vision Services

Clicks coordinates, types layouts, reads active nodes, and captures screens for the OCR models.

Voice Input & Wake Word

The entry point for hands-free local control. Monitors environmental audio offsets completely offline.

  • Porcupine Wake Word engine (offline listening)
  • Android native speech recognition (Speech-to-Text)
  • Seamless fallback: switches to manual chat input on speech failure
  • High fidelity TTS fallback with ElevenLabs API support
Developer Docs

Supported Action Intents

Below is a dynamic index of native and accessibility commands recognized by the Action Dispatcher and Planner engines.

🔍
Action Name Category Key Parameters Description / Execution Strategy
MAKE_CALL communication contact / phone Resolves contact with fuzzy matcher. Calls in background, falls back to native dialer if permissions fail.
SEND_SMS communication contact, message Attempts background SMS dispatch. Launches native compose intent on tablet/telephony failure.
SEND_WHATSAPP communication contact, message Deep-links into chat, uses Accessibility service node inspections to automate clicking "Send".
TOGGLE_FLASHLIGHT system on / off Binds to hardware CameraManager and flips flashlight state safely.
TOGGLE_WIFI system on / off Controls WifiManager settings to switch the Wi-Fi card state.
TOGGLE_BLUETOOTH system on / off Switches Bluetooth adapter states using Android Bluetooth settings API.
OPEN_APP system appName Resolves application query and fires launcher intents for target packages.
CLICK_TEXT accessibility text Traverses active accessibility view hierarchy node tree and clicks elements containing matching text.
CLICK_COORDINATES accessibility x, y Fires programmatic touch and click motions at specified coordinate layout parameters.
TYPE_TEXT accessibility targetText, content Finds an editable field, focuses, and injects string values using accessibility node APIs.
ANALYZE_SCREENSHOT accessibility - Captures screen buffer (WebP format), base64-encodes, and requests Vision LLM description. Falls back to text scraping on old APIs.
LIST_FILES files path Inspects directory contents at target path and returns list structure.
READ_FILE files path Loads file contents into memory stream, returning plain text with a size limit safety constraint.
WRITE_FILE files path, content Writes content to local text document. Resolves directory creation if paths don't exist.
